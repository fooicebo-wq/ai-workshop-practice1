CORE FRAMEWORK

知識庫跟資料庫，
最困難的是兩件事。

先想清楚一件事

這裡帶到一個
mindset

MD 是資料層：乾淨、可讀、可被 AI 穩定引用。Word、Excel、PDF、HTML 是表現層：給人看、給人操作、給人理解。所以不是哪個格式比較高級，而是什麼階段用什麼格式。

TWO LIBRARIES

公司要分清楚：
資料庫放事實，知識庫放經驗。

DATABASE

資料庫

客戶、訂單、合約、產品、員工等事實。

  • 放系統 / CRM / ERP
  • 可共用、可權限、可串工作流
KNOWLEDGE BASE

知識庫

設計原理、SOP、獎金辦法、銷售話術等經驗。

  • 放 Markdown / KB
  • 可搜尋、可引用、可成為 Skill 背景

CORE FRAMEWORK

公司，資料跟知識兩種庫。

資料庫放「事實」；知識庫放「經驗」。先分清楚，AI 才知道要去哪裡找答案。
DATABASE資料庫公司正在發生的事實，通常來自系統、表單或交易流程。
DB
客戶.md公司真的有哪一個客戶。
DB
連絡人.md誰負責、電話、角色與關係。
DB
產品.md / 報價單.md品項、規格、價格與交易事實。
KNOWLEDGE BASE知識庫公司怎麼判斷與怎麼做，通常來自 SOP、制度與經驗整理。
KB
磁磚銷售概要.md怎麼解釋產品，怎麼處理異議。
KB
SPC 鋪設辦法.md施工方法、注意事項與標準。
KB
新人手冊 / 獎金辦法.md公司制度、角色期待與工作規則。
判斷原則：會改變的事實放資料庫；要重複引用的經驗放知識庫。

SKILL PRACTICE

Skill，就是一個固定 SOP 動作。

練習不是寫程式，而是把「人怎麼做」整理成 AI 可以重跑的流程。
STEP 01先做出報價單 Skill
指令幫我生成一個報價單的 skill，使用這個模板（path）。
先固定格式、欄位與輸出規則，讓每次產出的報價單一致。
+
STEP 02再補上參照資料
指令報價單備註另外參照（path）。
Skill 不只要知道格式，也要知道內容來源；模板管輸出，參照管判斷。
練習目標：把一次人話指令，整理成可以重複使用、可追蹤、可修改的公司 SOP。

WHEN TO BUILD A SKILL

Skill 的重點，
不是炫技，是降低重複成本。

  • 01
    重複兩次的工作就可以開 skill
    不要等到流程完美才整理，先把可重複部分固定下來。
  • 02
    前情提要與背景設定也可以進 skill
    讓 AI 每次都用同一個角色、規則與資料來源工作。
  • 03
    Skill 可以有連續性
    上一個步驟的輸出，能成為下一個步驟的輸入。
  • 04
    Skill 可以套 skill
    大流程由小技能組成，像公司裡的部門協作。

SYSTEM FOUNDATION

CRM / ERP 是資料庫，
不是單純表格。

CONTENT

它存的不是檔案

客戶、聯絡人、產品、報價單、合約、採購與發票都會有關聯。

  • 資料一致
  • 可追蹤歷史
  • 可成為 AI 查詢來源
CAPABILITY

它必須具備三件事

能共用、有工作流、有權限，才適合接到 AI。

  • 共用：多人看到同一份真相
  • 流程：狀態能往下一步走
  • 權限：誰能看、誰能改要清楚

PRACTICE 5

練習五：新增 MCP 功能，並設定 MCP。

01
決定工具要做什麼
例如查客戶資料、查庫存、建立報價單。
02
把資料出口包起來
用 API / MCP 讓 AI 不直接亂碰資料庫。
03
設定工具說明與參數
AI 要知道何時使用、要填哪些欄位。
04
測試與加權限
能查不代表能改；能改也要記錄是誰改的。
